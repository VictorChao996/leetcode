# 說明學習重點

## 判斷今天是哪一天

如果 `$ARGUMENTS` 為空（使用者未指定天數）：

1. 讀取 `README.md`，找到「進度追蹤」區段
2. 找到最後一個 `- [x]` 的 Day，其**下一天**就是今天
3. 如果所有 Day 都未勾選，則今天是 Day 1
4. 將推算出的天數帶入後續步驟，並在輸出開頭告知使用者：「根據進度，今天是 Day X」

如果 `$ARGUMENTS` 有值，直接使用該值。

以下用 `N` 代表最終確定的天數。

## 讀取計畫

讀取 `7-day-sprint/dayN/plan.md`，擷取：

- 標題（Day N 的主題）
- 練題表格（題號、題目名、難度、類型、目的）
- 重點 API 與目標

## 檢查實際進度

1. 列出 `7-day-sprint/dayN/` 目錄下的所有檔案
2. **同時搜尋專案根目錄**的 `.py` 檔案（如 `704.binary-search.py`），這些也算已完成，但需提醒使用者搬移
3. 如果 `plan.md` 中已有 `## Completion Report`，讀取其中的狀態
4. 將計畫中的每道題與實際檔案交叉比對（優先看 `7-day-sprint/dayN/`，再看根目錄）：
   - 有對應的 `.py` 檔案 → ✅ 已完成
   - 檔案在根目錄而非 `dayN/` → ✅ 已完成，但標註 `📂 在根目錄`
   - 沒有對應檔案 → ⬜ 尚未開始
5. 統計完成數 / 總題數
6. 如果有任何檔案位於根目錄，在輸出末尾加上提醒：
   ```
   📂 以下檔案在專案根目錄，記得搬到 7-day-sprint/dayN/：
   - <檔名> → 7-day-sprint/dayN/
   ```

## 輸出格式

根據進度狀態，使用以下對應格式：

### 狀態 A：尚未開始（完成數 = 0）

```
📖 Day N：<標題>

🎯 今日重點
- （從 plan.md 的「重點 API」和「目標」擷取，每項一句話）

✍️ 練習清單
1. ⬜ <題號> <題名> (<難度>) — <目的>
2. ⬜ ...
（全部列出）

💡 建議順序
- （根據題目的預估時間和難度，建議從哪題開始）

⚠️ 注意事項
- （根據題型判斷容易踩坑或面試常問的地方，2-3 點）
```

### 狀態 B：進行中（0 < 完成數 < 總題數）

```
📖 Day N：<標題>（進度 M/T）

✅ 已完成
1. ✅ <題號> <題名> — <簡述解法>（從 Completion Report 擷取）
2. ✅ ...

⬜ 待完成
3. ⬜ <題號> <題名> (<難度>) — <目的>
4. ⬜ ...

💡 接下來建議
- 先做 <下一題建議>，原因：...

⚠️ 注意事項
- （針對「待完成」題目的提醒，2-3 點）
```

### 狀態 C：全部完成

```
📖 Day N：<標題>（✅ 全部完成）

✅ 完成總覽（M/T 題）
1. ✅ <題號> <題名> — <簡述解法>
2. ✅ ...

> Bonus: （如有額外解法檔案，列出）

🎉 Day N 已完成！可以用 /check-day N 產生完整報告，或繼續下一天。
```

## 規則

- 不要貼出完整程式碼，只做摘要
- 每個項目用一句話說明，保持精簡
- 注意事項應聚焦在「容易搞混」或「實務上重要」的點
- 題目比對用題號匹配（如 `704.*.py`），不要求完全檔名相同
- 如果目錄下有同一題號的多個檔案（如 `215.*.py` 有 3 個），最多列一個在主清單，其餘歸入 Bonus
- 總輸出控制在 25 行以內
